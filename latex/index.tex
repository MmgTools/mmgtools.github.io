Mmg provides 3 applications and 4 libraries\+:
\begin{DoxyItemize}
\item the {\bfseries{mmg2d}} application and library\+: mesh generation from a set of edges, adaptation and optimization of a bidimensionnal triangulation and isovalue discretization;
\item the {\bfseries{mmgs}} application and library\+: adaptation and optimization of a surface triangulation and isovalue discretization;
\item the {\bfseries{mmg3d}} application and library\+: adaptation and optimization of a tetrahedral mesh, isovalue discretization and lagrangian movement;
\item the {\bfseries{mmg}} library gathering the {\bfseries{mmg2d}}, {\bfseries{mmgs}} and {\bfseries{mmg3d}} libraries.
\end{DoxyItemize}\hypertarget{index_autotoc_md2}{}\doxysubsection{Get and compile the mmg project}\label{index_autotoc_md2}
\hypertarget{index_autotoc_md3}{}\doxysubsubsection{Needed tools}\label{index_autotoc_md3}
To get and build Mmg, you will need\+:
\begin{DoxyItemize}
\item {\bfseries{Git}}\+: to download the code you will have to use a git manager. You can install a git manager from the link below but there are many other git clients that you can use\+:
\begin{DoxyItemize}
\item \href{https://git-scm.com/download}{\texttt{ Official Git client}} (command line program)
\item \href{https://www.gitkraken.com/}{\texttt{ Git\+Kraken}}
\item \href{https://www.sourcetreeapp.com/}{\texttt{ Source\+Tree}} ~\newline

\end{DoxyItemize}

Note that if you uses Microsoft Visual Studio (Windows OS), you can simply activate the Git Module of the application.
\item {\bfseries{CMake}} \+: Mmg uses the CMake building system that can be downloaded on the following web page\+: \href{https://cmake.org/download/}{\texttt{ https\+://cmake.\+org/download/}}. On Windows OS, once CMake is installed, please  do not forget to mark the option\+: \`{}\`{}\`{} Add CMake to the system PATH for all users \`{}\`{}\`{}  ~\newline

\end{DoxyItemize}\hypertarget{index_autotoc_md4}{}\doxysubsubsection{Mmg download and compilation}\label{index_autotoc_md4}
\hypertarget{index_autotoc_md5}{}\doxyparagraph{Unix-\/like OS (\+Linux, Mac\+OS...)}\label{index_autotoc_md5}

\begin{DoxyEnumerate}
\item Get the repository\+: ~\newline
 
\begin{DoxyCode}{0}
\DoxyCodeLine{wget https://github.com/MmgTools/mmg/archive/master.zip}

\end{DoxyCode}
 or 
\begin{DoxyCode}{0}
\DoxyCodeLine{git clone https://github.com/MmgTools/mmg.git}

\end{DoxyCode}

\end{DoxyEnumerate}

The project sources are available under the {\bfseries{src/}} directory, see\+:
\begin{DoxyItemize}
\item {\bfseries{src/mmg2d/}} for files related to the mmg2d application;
\item {\bfseries{src/mmgs/}} for files related to the mmgs application;
\item {\bfseries{src/mmg3d/}} for files related to the mmg3d application;
\item {\bfseries{src/common/}} for files related to the both.
\end{DoxyItemize}

Fast compilation (build both {\bfseries{mmg2d}}, {\bfseries{mmgs}}, {\bfseries{mmg3d}}, the mmg2d static library ({\bfseries{libmmg3d.\+a}}), the mmgs static library ({\bfseries{libmmgs.\+a}}), the mmg3d static library ({\bfseries{libmmg3d.\+a}}) and the mmg static library ({\bfseries{libmmg.\+a}}))\+: ~\newline
 
\begin{DoxyCode}{0}
\DoxyCodeLine{cd mmg  }
\DoxyCodeLine{mkdir build  }
\DoxyCodeLine{cd build  }
\DoxyCodeLine{cmake ..  }
\DoxyCodeLine{make  }
\DoxyCodeLine{make install}

\end{DoxyCode}


If the {\ttfamily make install} command fail, try to run the {\ttfamily sudo make install} command. If you don\textquotesingle{}t have root access, please refers to the \href{https://github.com/MmgTools/Mmg/wiki/Setup-guide\#iii-installation}{\texttt{ Installation section}} of the \href{https://github.com/MmgTools/Mmg/wiki/Setup-guide\#setup-guide}{\texttt{ setup guide}}.

The {\bfseries{mmg2d}}, {\bfseries{mmgs}} and {\bfseries{mmg3d}} applications are available under the {\ttfamily mmg2d\+\_\+\+O3}, {\ttfamily mmgs\+\_\+\+O3} and {\ttfamily mmg3d\+\_\+\+O3} commands.

Note that if you use some specific options and want to set it easily, you can use a shell script to execute the previous commands. An example is provided \href{https://github.com/MmgTools/mmg/wiki/Configure-script-for-CMake-(UNIX-like-OS)}{\texttt{ here}}.\hypertarget{index_autotoc_md6}{}\doxyparagraph{Windows OS}\label{index_autotoc_md6}
The following compilation can be performed in any modern version of {\itshape Windows} (AKA 7, 8, 8.\+1 and 10). A basic knowledge of Windows is assumed (execute commands in cmd, create directories, etc...).

 \doxysubparagraph*{Compile with Visual\+Studio}

Universal windows platform development
\begin{DoxyEnumerate}
\item Get the {\bfseries{Visual Studio}} software\+: it can be downloaded \href{https://www.visualstudio.com/downloads/}{\texttt{ here}};
\item if not done during the previous step, download {\bfseries{C/\+C++}} compilers\+: in the Visual Studio searching zone, search {\bfseries{C compiler}} and install the {\bfseries{Visual C++ compilers and libraries}} (individual componant) and the MSBuild componant;
\item in the Visual Studio searching zone, search the {\bfseries{git}} word and select the installation of the {\bfseries{Git\+Hub extension for Visual\+Studio}};
\item stay in Visual\+Studio and clone the Mmg repository from the following url\+: \href{https://github.com/MmgTools/mmg.git;}{\texttt{ https\+://github.\+com/\+Mmg\+Tools/mmg.\+git;}}
\item Use {\bfseries{CMake}} to configure and generate your project. It can be done either with the graphic mode of CMake (you have to select the \char`\"{}\+Visual\+Studio\char`\"{} generator) or with a command line. In this case, it is highly recommended to specify that you intent to build a Visual\+Studio project. For example, if you are using Visual\+Studio 2017\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{cmake -\/G "{}Visual Studio 15 2017 Win64"{} \string^}
\DoxyCodeLine{configure}

\end{DoxyCode}

\end{DoxyEnumerate}

Note that you can use a script to make this step easier (an example of script is provided \href{https://github.com/MmgTools/mmg/wiki/Configure-script-for-CMake-(Windows-OS)}{\texttt{ here}}). ~\newline


Once the configuration script has finished without errors a {\ttfamily mmg.\+sln} file will be generated in the cmake\+\_\+build directory.


\begin{DoxyEnumerate}
\item Double click this file and the visual studio project will open. Then choose the project configuration (Release, Debug...). ~\newline
 Please, make sure that the project is set to Win32 or x64 and change it if is not. Finally, in order to compile Mmg, right click the {\ttfamily INSTALL} project and select the option {\ttfamily BUILD}.
\end{DoxyEnumerate}

 \doxysubparagraph*{Compile with Min\+GW}


\begin{DoxyEnumerate}
\item Get a {\bfseries{C Compiler}}\+:
\begin{DoxyItemize}
\item {\bfseries{Min\+GW}} can be downloaded \href{http://mingw.org/}{\texttt{ here}}. We recommand to install the {\itshape mingw-\/developer-\/tools}, {\itshape mingw32-\/base}, {\itshape mingw32-\/gcc-\/fortran}, {\itshape mingw32-\/gcc-\/g++} and {\itshape msys-\/base} packages;
\item Edit the environment variables and add Min\+GW in your {\bfseries{PATH}} variable. It can be done in the {\bfseries{advanced system settings}} panel. (note that you must modify the {\bfseries{PATH}} variable, not the {\bfseries{Path}} one);
\item {\bfseries{Min\+GW}} binaries are probably in {\ttfamily C\+:\textbackslash{}Min\+GW\textbackslash{}bin}
\item the Min\+GW terminal is in {\ttfamily C\+:\textbackslash{}Min\+GW\textbackslash{}msys\textbackslash{}1.\+0\textbackslash{}msys}
\end{DoxyItemize}
\item Clone the Mmg repository from the following url\+: \href{https://github.com/MmgTools/mmg.git;}{\texttt{ https\+://github.\+com/\+Mmg\+Tools/mmg.\+git;}}
\item Quit and restart the {\itshape CMake} application to take the PATH modification into account then use CMake to configure and generate your project (select the Min\+GW Makefiles generator of CMake). If you have installed the scotch libraries, you will need to set explicitely the libraries paths;
\item Build the Mmg applications\+: in the min\+GW prompt ({\ttfamily C\+:\textbackslash{}Min\+GW\textbackslash{}msys\textbackslash{}1.\+0\textbackslash{}msys}) run\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{mingw32-\/make}

\end{DoxyCode}

\end{DoxyEnumerate}

Again, if you use some specific options and want to make the CMake configuration step easier, you can use a batch script.
