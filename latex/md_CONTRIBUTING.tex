\hypertarget{md_CONTRIBUTING_autotoc_md25}{}\doxysubsection{2) Where}\label{md_CONTRIBUTING_autotoc_md25}
Please, comments only your functions in the {\ttfamily .c} file, except for the {\bfseries{API}} functions; these must be documentated in the appropriate {\ttfamily libmmg\texorpdfstring{$<$}{<}X\texorpdfstring{$>$}{>}.h} file (and only there).\hypertarget{md_CONTRIBUTING_autotoc_md26}{}\doxysection{II/ Memory management\+: dynamic allocations and deallocations}\label{md_CONTRIBUTING_autotoc_md26}
We need to control the memory consumption in our applications so the memory used by dynamic allocations is counted and updated at each allocation and deallocation.

Note that with a high verbosity (at least 6), you can check that at the end of the process the memory count is 0.

To make the update of memory consumption easier, we have wrapped the {\ttfamily malloc}, {\ttfamily calloc}, {\ttfamily realloc} and {\ttfamily free} functions into macros that must be called in place of these functions.

\hypertarget{md_CONTRIBUTING_autotoc_md29}{}\doxysection{III/ Coding style}\label{md_CONTRIBUTING_autotoc_md29}
Please, use the following configuration in your editor\+:
\begin{DoxyItemize}
\item no tabs;
\item 1 indent = 2 spaces;
\item no trailing whitespaces;
\item limit the size of your lines to 80 characters;
\end{DoxyItemize}

Besides, try to respect the following rules\+:
\begin{DoxyItemize}
\item declaration of variables in the top of the function;
\item do not use exit(), use a return value instead;
\item do not implement void API functions;
\item the main library functions returns {\ttfamily MMG\texorpdfstring{$<$}{<}X\texorpdfstring{$>$}{>}\+\_\+\+SUCCESS} if successful, {\ttfamily MMG\texorpdfstring{$<$}{<}X\texorpdfstring{$>$}{>}\+\_\+\+LOWFAILURE} if the function fails but we can save the mesh, and {\ttfamily MMG\texorpdfstring{$<$}{<}X\texorpdfstring{$>$}{>}\+\_\+\+STRONGFAILURE} if fails and we can\textquotesingle{}t save a conforming mesh. 
\end{DoxyItemize}
