Initialize handling of multimaterial mode.

An indexed table of materials has been provided by the \mbox{\hyperlink{structMMG5__Mat}{MMG5\+\_\+\+Mat}} datatype in the form\+:\hypertarget{mmgcommon__private_8h_autotoc_md30}{}\doxysubsubsection{index $\vert$ dospl       $\vert$ ref       $\vert$ rin       $\vert$ rex}\label{mmgcommon__private_8h_autotoc_md30}
0 $\vert$ dospl\+\_\+0 $\vert$ ref\+\_\+0 $\vert$ rin\+\_\+0 $\vert$ rex\+\_\+0 ... $\vert$ ... $\vert$ ... $\vert$ ... $\vert$ k $\vert$ dospl\+\_\+k $\vert$ ref\+\_\+k $\vert$ rin\+\_\+k $\vert$ rex\+\_\+k ... $\vert$ ... $\vert$ ... $\vert$ ... $\vert$ n-\/1 $\vert$ dospl\+\_\+\{n-\/1\} $\vert$ ref\+\_\+\{n-\/1\} $\vert$ rin\+\_\+\{n-\/1\} $\vert$ rex\+\_\+\{n-\/1\}

where dospl is the split/preserve attribute of the material, and rin,rex are its child materials (if dospl). Viceversa, ref is the parent material for rin,rex.

Here a lookup table for material references is built through trivial hashing for all references (both parent and child materials) with the key\+:

key = ref -\/ ref\+\_\+min, ref\+\_\+min = min\+\_\+\{k = 0,n-\/1\} (ref\+\_\+k, rin\+\_\+k, rex\+\_\+k)

For all references, it is important to store
\begin{DoxyItemize}
\item the index k of the row in the original table,
\item the characteristic attribute (parent-\/split, parent-\/preserve, child-\/interior, child-\/exterior) of the material.
\end{DoxyItemize}

Since dospl = 0 or 1, MG\+\_\+\+PLUS = 2, and MG\+\_\+\+MINUS = 3

we can store the attribute as dospl (for the parent material) or MG\+\_\+\+MINUS/ MG\+\_\+\+PLUS (for the child materials), with its value ranging between 0 and 3.

A convenient entry to store both the index and the attribute in the lookup table is thus\+:

entry = 4$\ast$(index+1) + attribute

leading to a lookup table in the form (the key ordering is only an example)\+:\hypertarget{mmgcommon__private_8h_autotoc_md31}{}\doxysubsubsection{key   $\vert$ entry}\label{mmgcommon__private_8h_autotoc_md31}
... $\vert$ ... ref\+\_\+k $\vert$ 4$\ast$(k+1)+dospl\+\_\+k ... $\vert$ ... rin\+\_\+k $\vert$ 4$\ast$(k+1)+MG\+\_\+\+MINUS ... $\vert$ ... rex\+\_\+k $\vert$ 4$\ast$(k+1)+MG\+\_\+\+PLUS ... $\vert$ ...

where the index and the attribute of the material can be retrieved as

index = entry / 4 -\/1 attribute = entry \% 4

What if two materials have the same reference?
\begin{DoxyItemize}
\item child references should be distinct (the entry in the lookup table would be overwritten),
\item a parent material can have itself as child (a positive attribute would say it should be split, the attribute value would say if it is interior or exterior). Thus, each of the maps parent-\/\texorpdfstring{$>$}{>}child\+\_\+in and parent-\/\texorpdfstring{$>$}{>}child\+\_\+ext is injective, but a fixed point is allowed.
\end{DoxyItemize}

Why child materials should be different?
\begin{DoxyItemize}
\item because on failure it is necessary to recover the parent references and apply them on the tetrahedra to restore the starting materials distribution. 
\end{DoxyItemize}
